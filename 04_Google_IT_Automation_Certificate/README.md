# Phase 1: Core Python Syntax & Control Flow

## 🎯 Overview
This repository contains my consolidated project for Modules 1 & 2 of the Google IT Automation with Python program. Instead of writing isolated generic scripts, I have combined all the foundational concepts to build a **Clinical Triage Priority System**. This script automates the decision-making process in a hospital emergency ward based on patient vitals.

## 🧠 Core Concepts Implemented
Based on the Coursera curriculum, the following concepts have been successfully integrated into the logic:
* **Variables & Data Types:** Managing patient data (Strings, Integers, Booleans).
* **Expressions & Type Conversion:** Implicit and Explicit conversions (e.g., using `str()` for formatted outputs).
* **Defining Functions:** Modularizing the code with `def` and utilizing `return` values for cleaner logic.
* **Complex Branching:** Implementing `if`, `elif`, and `else` statements to create multiple clinical outcomes.
* **Operators:** Utilizing comparison (`<`, `>`, `==`) and logical operators (`and`, `or`) to evaluate multi-layered medical parameters.
* **Code Style:** Applying industry-standard self-documenting variables and clean comments for readability.

## 🚀 The Output (`patient_triage_system.py`)
The script evaluates parameters such as age, oxygen levels, and chest pain symptoms to automatically return a triage priority (CRITICAL, HIGH, MODERATE, or NORMAL), demonstrating how basic programming syntax can be leveraged for medical data automation.
